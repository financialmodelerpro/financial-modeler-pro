## Supabase Storage Buckets

| Bucket | Purpose | Access |
|--------|---------|--------|
| `certificates` | Certificate PDF templates + issued PDFs | Public |
| `badges` | Badge PNG templates + issued badges | Public |
| `course-materials` | Lesson/course file attachments (PDF, Word, PPT, Excel, images) | Public |
| `live-session-banners` | Live session banner images | Public |
| `cms-assets` | CMS uploaded media (images, logos) | Public |
| `marketing-assets` | Marketing Studio uploaded background images (PNG / JPEG / WebP, max 10 MB). Created by migration 142. Public-read SELECT policy so satori can fetch them when rendering. Files stored under `bg/<timestamp>-<rand>.<ext>`. Metadata indexed in `marketing_uploaded_assets`. | Public |
| `model-submissions` | Student-built financial models uploaded for admin review (migration 148). **Private** (no public SELECT policy). Files stored under `<email>/<course>/attempt<N>_<ts>_<safeName>.<ext>`. Allowed extensions: `xlsx`, `xls`, `xlsm`, `pdf`. Default 10 MB cap, configurable via `training_settings.model_submission_max_file_size_mb`. Admin reads via the proxy route `GET /api/admin/model-submissions/[id]/file` so the bucket can stay private. | Private |

## Database Tables (Supabase)

### Modeling Hub
| Table | Purpose |
|-------|---------|
| `users` | Accounts: id, email, name, password_hash, role, subscription_plan, subscription_status, phone, city, country, email_confirmed, confirmed_at |
| `projects` | Legacy generic JSON blob, all platforms (pre-M1.6). Superseded for REFM by `refm_projects` + `refm_project_versions` (migration 149). Other platforms (BVM/FPA/etc.) still use this table until they migrate. |
| `refm_projects` | REFM Module 1 project metadata (migration 149, Phase M1.6). One row per project, owned by exactly one user (`user_id UUID REFERENCES users(id) ON DELETE CASCADE`). Columns: `id UUID PK`, `user_id`, `name`, `location`, `status TEXT CHECK ('Draft','Active','IC Review','Approved','Archived')`, `asset_mix JSONB DEFAULT '[]'` (cached array of asset names for the picker tile), `schema_version INT DEFAULT 4` (snapshot shape version, currently v4 from M1.5/12), `current_version_id UUID REFERENCES refm_project_versions(id) ON DELETE SET NULL` (pointer to the version row that loads on open, bumped on every save), `created_at`, `updated_at` (auto-touched via `trg_refm_projects_touch_updated_at` trigger). Index `idx_refm_projects_user_updated (user_id, updated_at DESC)` covers the picker hot path. RLS enabled; policies use `auth.uid()` so anon-key clients deny-all (project uses NextAuth, not Supabase auth — server uses SERVICE_ROLE which bypasses RLS, and API routes under `app/api/refm/projects/*` enforce `user_id = session.user.id` on every query). |
| `refm_project_versions` | REFM snapshot history (migration 149, Phase M1.6). Append-only, never UPDATEd; save = INSERT new row + bump `refm_projects.current_version_id`. Columns: `id UUID PK`, `project_id UUID REFERENCES refm_projects(id) ON DELETE CASCADE`, `version_number INT CHECK >= 1` (monotonic per project, server reads MAX and writes MAX+1), `schema_version INT DEFAULT 4`, `snapshot JSONB NOT NULL` (full HydrateSnapshot from `module1-store.ts`, read-tolerant via `lib/state/module1-migrate`), `label TEXT NULL` (set by VersionModal save-as; auto-saves leave it NULL), `created_at`. Unique index `uniq_refm_versions_project_number (project_id, version_number)` enforces no concurrent-save dupes. Index `idx_refm_versions_project_created (project_id, created_at DESC)` for the version-list UI. Same RLS posture as `refm_projects` (transitive ownership through `project_id`). **Phase M1.7 (2026-05-02) extended the JSONB shape additively** — `snapshot.plots[]` and `snapshot.zones[]` carry the M1.7 area-program entities, and AssetClass / SubUnit gained optional `plotId / zoneId / primaryStrategy / mepPct / backOfHousePct / otherTechnicalPct / parkingBaysPerUnit / gfaSharePct` fields. Pre-M1.7 snapshots load via `enrichWithHierarchyDefaults` padding empty plots/zones; no migration needed (JSONB absorbs the new keys natively). |
| `password_resets` | Tokens for forgot-password flow |
| `modeling_email_otps` | 6-digit device-verify OTPs (10-min TTL) |
| `modeling_access_whitelist` | Pre-launch access allowlist (migration 136): `id UUID PK`, `email TEXT UNIQUE`, `note TEXT NULL`, `added_by TEXT NULL`, `added_at TIMESTAMPTZ DEFAULT NOW()`. Partial index `idx_modeling_wl_email_lower` on `LOWER(email)`. Admin pre-seeded. Consulted by `canEmailSigninModeling` / `canEmailRegisterModeling` in `src/hubs/modeling/lib/access.ts` so whitelisted emails bypass the signin + register Coming Soon toggles. Admin UI at `/admin/modeling-access` (add + revoke). |

### Certification Watch & Assessment Results
| Table | Purpose |
|-------|---------|
| `certification_watch_history` | Video watch status (in_progress/completed) + interval-merged tracking. Columns: `watch_seconds`, `total_seconds`, `watch_percentage`, `last_position`, `updated_at` (mig 103); **`watch_intervals JSONB DEFAULT '[]'`** (mig 146 - canonical multi-session accumulator. Client posts a snapshot of `[start,end]` tuples on every progress tick; server unions with existing JSONB and re-derives `watch_seconds = sumIntervals(merged)` so cross-session coverage actually accumulates. Null on in_progress + legacy rows); **`video_load_at TIMESTAMPTZ NULL`** (mig 147 - server-stamped on first progress POST per row. Anchors the manual override elapsed-time check: `now - video_load_at >= total_seconds * 0.8`). Gates "Take Assessment" on dashboard + Mark Complete on watch page. `student_email + tab_key` UNIQUE. |
